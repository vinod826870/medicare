## Database Schema Comparison

### âŒ Old Schema (Broken)

```sql
CREATE TABLE cart_items (
  id uuid PRIMARY KEY,
  user_id uuid REFERENCES profiles(id),
  medicine_id uuid REFERENCES medicines(id),  -- âŒ PROBLEM!
  quantity integer,
  created_at timestamptz,
  updated_at timestamptz
);

-- medicines table was DELETED in migration 00002
-- But foreign key constraint still exists!
-- Result: All inserts FAIL
```

### âœ… New Schema (Fixed)

```sql
CREATE TABLE cart_items (
  id uuid PRIMARY KEY,
  user_id uuid REFERENCES profiles(id),
  medicine_id text NOT NULL,  -- âœ… FIXED!
  quantity integer CHECK (quantity > 0),
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now(),
  UNIQUE(user_id, medicine_id)
);

-- No foreign key constraint on medicine_id
-- Can store ANY medicine ID from external APIs
-- Result: All inserts SUCCEED
```
